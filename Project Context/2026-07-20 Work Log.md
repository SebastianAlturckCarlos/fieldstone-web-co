## What was built

### Backend (`apps/backend`)
- `src/core/brand_extract.ts` (new) — deterministic brand parsing from raw homepage HTML: theme-color meta wins, else stylesheet hex-frequency ranking (grey/near-black/near-white filtered, near-duplicates deduped); logo from `<img>` self-identifying as logo → og:logo → apple-touch-icon → og:image, resolved absolute.
- `src/core/mockup_template.ts` (new) — dependency-free HTML render of the Tier-1 dispatch screen, themed by CSS vars from the captured brand, **trade-aware sample jobs** (HVAC / plumbing / electrical pools), audit's top flaw answered in the lead metric ("After-Hours Calls Captured" when the top flaw is conversion/mobile). Falls back to Fieldstone blue + company-initials badge when a site yields nothing.
- `src/core/snapshot.ts` (new) — puppeteer-core against the machine's own Chrome/Edge (auto-detected, `SNAPSHOT_BROWSER_PATH` override; no bundled-Chromium download). 1360×760 @ DPR 2. **Never a gate**: any failure logs once and the email ships text-only.
- `site_fetch.ts` — returns `brand` alongside the audit excerpt; excerpt limit 9,000 → 32,000 chars.
- `orchestrator.ts` — stores `brand_json` on the lead, renders the snapshot after the consult pass, stores `snapshot_path` on the outreach row. CMO/consult payloads now carry a `preview` object instead of `demo_link` (`demo_link.ts` deleted).
- `send.ts` — `deliver()` now sends `text` + `html` alternatives; HTML embeds the snapshot via `attachments[{content, content_id}]` + `<img src="cid:fieldstone-preview">`, deliberately plain markup (designed-looking cold email gets filtered). Compliance footer on both parts, exactly once. Follow-ups (`replies.ts`) re-attach the preview as visual proof — the thing the Sprint-3 comment said belonged there.
- `server.ts` — new `GET /api/leads/{id}` (drill-down: audit, brand, drafts+consults, agent runs, episodic memory) and `GET /api/snapshots/{leadId}.png`; approvals now include `has_snapshot` + parsed `brand`.
- `profiles.json` — CMO/CEO/consult prompts rewritten for the snapshot era (reference the embedded preview exactly once, never a URL; CEO rubric now scores the preview reference; researcher pushed to a 4–6-flaw thorough audit). max_tokens: CEO 2048, CMO 1536, SLS/consult 1024, ANL 2048.
- Schema/migrations: `leads.brand_json`, `outreach_emails.snapshot_path` (idempotent `ensureColumn`).
- Deps: `puppeteer-core` added; `@types/node` + `@types/better-sqlite3` as devDeps (backend now typechecks clean with `tsc --noEmit`).
